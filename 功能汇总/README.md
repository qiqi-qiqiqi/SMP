# 智能门禁系统 - 项目介绍

## 一、项目概述

本项目是一个基于树莓派3B的智能门禁系统，支持多种开锁方式（人脸、密码、指纹、卡片），并集成了环境监测（温湿度、雨量）和华为云IoT平台数据上云功能。

### 功能特性

| 功能模块 | 功能描述 |
|---------|---------|
| 人脸开锁 | 基于深度学习的人脸识别，支持人脸采集、训练和实时识别 |
| 密码开锁 | 支持用户自定义密码验证 |
| 指纹开锁 | 预留指纹模块接口（待扩展） |
| 卡片开锁 | 预留RFID卡片接口（待扩展） |
| 环境监测 | 温湿度（DHT11）、雨量传感器、光照强度 |
| 远程控制 | 华为云IoT平台指令下发，支持继电器和蜂鸣器控制 |
| 数据上云 | 定时上报环境数据和开门事件 |

## 三、人脸识别原理详解

### 3.1 技术选型

本项目使用 **OpenCV DNN 模块** 结合两个预训练模型实现人脸识别：

| 模型 | 文件 | 功能 |
|------|------|------|
| YuNet | `face_detection_yunet_2023mar.onnx` | 人脸检测，输出人脸边界框和关键点 |
| SFace | `face_recognition_sface_2021dec_int8.onnx` | 人脸特征提取，输出128维特征向量 |

### 3.2 人脸识别流程

#### 阶段一：人脸采集

```python
# 核心流程：采集照片 → 保存到 dataset/{姓名}/ 文件夹
def capture_faces(self, name):
    # 1. 创建保存目录
    save_dir = os.path.join(DATASET_ROOT, name)
    # 2. 打开摄像头，循环显示画面
    cap = self._get_camera()
    # 3. 鼠标点击拍照，保存为 {序号}.jpg
    cv2.imwrite(save_path, frame)
```

**采集说明**：
- 为每个人创建独立文件夹（如 `dataset/W/`）
- 建议采集20-100张不同角度、光照的照片
- 照片命名格式：`0.jpg`, `1.jpg`, `2.jpg`...

#### 阶段二：特征库训练

训练流程分为三个步骤：

```python
def build_database(self):
    # 步骤1：人脸检测 → 获取人脸边界框
    _, faces = self.detector.detect(img)
    
    # 步骤2：人脸对齐 → 标准化人脸姿态
    aligned = self.recognizer.alignCrop(img, faces[0])
    
    # 步骤3：特征提取 → 生成128维特征向量
    feat = self.recognizer.feature(aligned).flatten()
    feat = feat / np.linalg.norm(feat)  # L2归一化
    
    # 步骤4：特征平均 → 对同一人多张照片取平均
    avg = np.mean(feats, axis=0)
```

**关键技术点**：

| 步骤 | 技术原理 | 作用 |
|------|---------|------|
| 人脸检测 | YuNet 轻量级人脸检测网络 | 定位图像中所有人脸的位置 |
| 人脸对齐 | 基于关键点的仿射变换 | 将人脸旋转、缩放至标准姿态 |
| 特征提取 | SFace 人脸识别网络 | 将人脸转换为128维特征向量 |
| L2归一化 | `feat / ||feat||` | 消除光照、距离对特征的影响 |
| 特征平均 | 多张照片特征取平均 | 提升识别鲁棒性 |

#### 阶段三：实时识别

识别流程：

```python
def recognize(self, frame):
    # 1. 人脸检测
    _, faces = self.detector.detect(frame)
    
    # 2. 人脸对齐 + 特征提取
    aligned = self.recognizer.alignCrop(frame, face)
    feat = self.recognizer.feature(aligned).flatten()
    feat = feat / np.linalg.norm(feat)
    
    # 3. 余弦相似度计算
    sims = np.dot(self.db_features, feat)
    
    # 4. 阈值判断
    if max_sim >= SIM_THRESHOLD:
        return db_names[max_idx], face_location
    else:
        return "Unknown", []
```

### 3.3 余弦相似度原理

**核心公式**：

```
相似度 = cos(θ) = (A · B) / (||A|| × ||B||)
```

其中：
- `A`：待识别人脸的128维特征向量
- `B`：特征库中某个人的平均特征向量
- `A · B`：向量点积
- `||A||`：向量L2范数（长度）

**相似度含义**：

| 相似度值 | 含义 | 判定结果 |
|---------|------|---------|
| 0.9+ | 极高相似度，几乎是同一人 | 识别成功 |
| 0.7-0.9 | 较高相似度，可能是同一人 | 根据阈值判定 |
| 0.5-0.7 | 中等相似度，可能是相似的人 | 通常判定为未知 |
| < 0.5 | 低相似度，不同的人 | 未知 |

**项目配置**：`SIM_THRESHOLD = 0.65`（可在 `config.py` 中调整）

### 3.4 模型文件说明

| 文件 | 大小 | 来源 | 说明 |
|------|------|------|------|
| `face_detection_yunet_2023mar.onnx` | ~1.3MB | OpenCV官方模型库 | 轻量级人脸检测，适合边缘设备 |
| `face_recognition_sface_2021dec_int8.onnx` | ~3.5MB | OpenCV官方模型库 | 量化后的SFace模型，速度更快 |

**模型获取**：可从 [OpenCV Model Zoo](https://github.com/opencv/opencv_zoo) 下载最新版本。

### 6.4 人脸识别流程

1. **用户点击"人脸验证"** → 系统拍摄一张照片
2. **人脸检测** → YuNet模型检测照片中的人脸
3. **特征提取** → SFace模型提取128维特征向量
4. **相似度比对** → 与特征库中的所有人计算余弦相似度
5. **结果判定** → 超过阈值则识别成功，自动开门
6. **自动关门** → 3秒后自动关闭继电器

## 八、常见问题

### 8.1 摄像头打开失败

```
[ WARN:0] global cap_v4l.cpp:999 open VIDEOIO(V4L2:/dev/video0): can't open camera by index
```

**解决方法**：
1. 检查摄像头连接：`ls /dev/video*`
2. 修改 `config.py` 中的 `CAM_ID`
3. 确保摄像头权限：`sudo chmod 666 /dev/video0`

### 8.2 人脸识别准确率低

**解决方法**：
1. 增加训练照片数量（建议20张以上）
2. 采集不同角度、光照的照片
3. 调整 `SIM_THRESHOLD` 阈值（降低阈值更宽松）

### 8.3 雨量传感器始终显示有雨

**检查步骤**：
1. 查看终端调试信息：`[DEBUG] 雨量传感器引脚 17: 高电平`
2. 检查传感器接线是否正确
3. 根据实际电平逻辑调整传感器

### 8.4 MQTT连接失败

**解决方法**：
1. 检查网络连接：`ping e201e15730.st1.iotda-device.cn-north-4.myhuaweicloud.com`
2. 确认华为云设备证书信息正确
3. 检查端口是否开放：8883（TLS）

## 九、代码优化建议

### 9.1 当前优化

| 优化项 | 实现方式 | 效果 |
|--------|---------|------|
| 人脸检测线程 | 使用 `threading.Thread` | 避免阻塞UI主线程 |
| 传感器数据缓存 | 保存上次成功读取的值 | 避免读取失败时返回None |
| MQTT重连机制 | 自动重试连接 | 保证网络断开后自动恢复 |
| 低功耗模式 | 降低摄像头分辨率 | 减轻树莓派CPU负担 |

### 9.2 进一步优化方向

1. **人脸特征增量更新**：支持不重新训练添加新用户
2. **活体检测**：防止照片欺骗攻击
3. **模型量化**：使用更轻量级的模型提升速度
4. **本地缓存**：网络断开时本地存储数据，恢复后补传
