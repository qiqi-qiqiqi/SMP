# 智慧门禁系统 - 上位机框架说明文档

## 一、项目概述

本项目是基于 **Python Tkinter** 构建的智慧门禁系统上位机界面框架，运行于树莓派平台，支持多种开锁方式（人脸、密码、指纹、卡片）和设备管理功能。框架采用 **控制器-页面** 的MVC模式，实现了模块化设计和跨平台调试能力。

## 三、核心实现方法

### 3.1 控制器-页面模式

代码位置：[gui_main.py](file:///f:/book/就业/项目/树莓派/门禁/上位机框架/gui_main.py#L12-L53)

```python
class SmartDoorApp(tk.Tk):
    def __init__(self):
        super().__init__()
        self.title("智慧门禁系统")
        self.geometry("800x480")
        self.attributes("-fullscreen", True)

        # 初始化服务对象（注入到所有页面）
        self.sensors = Sensors()
        self.pwd_manager = PasswordManager()
        self.face_recognizer = FaceRecognizer()

        # 创建页面容器
        container = tk.Frame(self)
        container.pack(side="top", fill="both", expand=True)
        container.grid_rowconfigure(0, weight=1)
        container.grid_columnconfigure(0, weight=1)

        # 注册所有页面（页面只创建一次，切换时复用）
        self.frames = {}
        for PageClass in (HomePage, PasswordPage, FingerprintPage, CardPage, AdminPage):
            page_name = PageClass.__name__
            frame = PageClass(parent=container, controller=self)
            self.frames[page_name] = frame
            frame.grid(row=0, column=0, sticky="nsew")

        self.show_frame("HomePage")
```

**设计特点**：

1. **单例页面**：所有页面在初始化时创建，通过 `tkraise()` 切换显示，避免重复创建开销
2. **依赖注入**：`sensors`、`pwd_manager`、`face_recognizer` 在控制器中初始化，通过 `controller` 参数传递给各页面
3. **统一容器**：所有页面共享同一个 `container`，通过 `grid` 布局堆叠

### 3.2 页面切换机制

代码位置：[gui_main.py](file:///f:/book/就业/项目/树莓派/门禁/上位机框架/gui_main.py#L39-L47)

```python
def show_frame(self, page_name):
    # 停止当前页面的摄像头（资源管理）
    if self.current_page and hasattr(self.frames[self.current_page], 'stop_camera'):
        self.frames[self.current_page].stop_camera()
    frame = self.frames[page_name]
    frame.tkraise()  # 提升到最顶层
    self.current_page = page_name
    if hasattr(frame, 'on_show'):
        frame.on_show()  # 页面显示回调
```

**页面生命周期**：

```
创建 (__init__) → 隐藏 (grid堆叠) → 显示 (on_show) → 交互 → 隐藏 (stop_camera) → 销毁 (cleanup)
```

| 生命周期钩子 | 作用 | 实现位置 |
|-------------|------|----------|
| `__init__` | 页面初始化，创建UI组件 | 各页面类 |
| `on_show` | 页面显示时的初始化操作（启动摄像头、重置状态） | 各页面类（可选） |
| `stop_camera` | 页面隐藏时的资源释放 | `HomePage` |
| `cleanup` | 程序退出时的全局清理 | `SmartDoorApp` |

### 3.4 摄像头实时显示实现

代码位置：[pages/home_page.py](file:///f:/book/就业/项目/树莓派/门禁/上位机框架/pages/home_page.py#L116-L138)

```python
def update_video(self):
    if not self.camera_running:
        return
    ret, frame = self.cap.read()
    if ret:
        # 隔帧识别，减轻CPU负担
        if self.face_recognition_active and self.frame_count % 3 == 0:
            name, face_locations = self.controller.face_recognizer.recognize(frame)
            if face_locations:
                for (top, right, bottom, left) in face_locations:
                    cv2.rectangle(frame, (left, top), (right, bottom), (0,255,0), 2)
                    cv2.putText(frame, name, (left, top-10),
                                cv2.FONT_HERSHEY_SIMPLEX, 0.8, (0,255,0), 2)
        # OpenCV BGR → PIL RGB → ImageTk PhotoImage
        frame_rgb = cv2.cvtColor(frame_resized, cv2.COLOR_BGR2RGB)
        img = Image.fromarray(frame_rgb)
        imgtk = ImageTk.PhotoImage(image=img)
        self.video_label.imgtk = imgtk  # 防止GC回收
        self.video_label.configure(image=imgtk)
    self.frame_count += 1
    self.after_id = self.after(80, self.update_video)  # ~12fps
```

**技术要点**：

1. **格式转换链**：`OpenCV (BGR)` → `PIL Image (RGB)` → `ImageTk.PhotoImage`
2. **内存管理**：`self.video_label.imgtk = imgtk` 防止Tkinter的垃圾回收机制删除图片对象
3. **帧率控制**：`self.after(80, self.update_video)` 设置约12fps的刷新率
4. **隔帧识别**：`self.frame_count % 3 == 0` 每3帧进行一次人脸识别，降低CPU负载

## 四、人脸识别实现对比

### 4.1 两种实现方式对比

本项目包含两种人脸识别实现方式，分别位于不同目录：

| 维度 | 上位机框架 (face_recognition库) | 人脸识别文件夹 (OpenCV SFace) |
|------|---------------------------------|------------------------------|
| **库依赖** | `face_recognition` (dlib) | `opencv-python` |
| **特征维度** | 128维 (dlib模型) | 128维 (SFace模型) |
| **模型文件** | 无需额外模型文件 | `sface.onnx`, `yunet.onnx` |
| **特征库格式** | 单张图片=1个特征 | 多张图片→增强→多个特征聚合 |
| **匹配方式** | `compare_faces()` (欧氏距离) | 余弦相似度 + Top-K平均 |
| **数据增强** | 无 | 翻转、旋转、亮度/对比度调整 |
| **部署难度** | dlib编译复杂 | OpenCV安装简单 |
| **性能** | 较慢 | 较快（INT8量化） |

### 4.2 上位机框架中的实现

代码位置：[face_recognizer.py](file:///f:/book/就业/项目/树莓派/门禁/上位机框架/face_recognizer.py#L94-L110)

```python
def recognize(self, frame):
    if self.mock:
        # 模拟模式：随机返回已知人脸或Unknown
        if random.random() > 0.5 and self.known_names:
            return random.choice(self.known_names), [(100, 200, 200, 100)]
        return "Unknown", []
    
    # 真实模式：face_recognition库
    rgb = cv2.cvtColor(frame, cv2.COLOR_BGR2RGB)
    locations = face_recognition.face_locations(rgb)  # 人脸检测
    encodings = face_recognition.face_encodings(rgb, locations)  # 特征提取
    
    for encoding, loc in zip(encodings, locations):
        matches = face_recognition.compare_faces(self.known_encodings, encoding, 
                                                  tolerance=FACE_MATCH_THRESHOLD)
        if True in matches:
            idx = matches.index(True)
            return self.known_names[idx], locations
    return "Unknown", locations
```

**工作流程**：

```
已知人脸图片 → load_known_faces() → known_encodings + known_names
                                              ↓
实时帧 → face_locations() → 人脸位置 → face_encodings() → 待识别特征
                                              ↓
compare_faces() → 欧氏距离比对 → 阈值判断 → 返回姓名
```

## 五、页面功能详解

### 5.1 主页（HomePage）

**功能**：系统主界面，提供开锁方式入口和环境状态显示

**核心组件**：
- 视频显示区：实时摄像头画面，支持人脸验证开关
- 功能按钮：人脸验证、密码开锁、指纹开锁、卡片开锁、管理员
- 状态栏：温度、湿度、光照、继电器状态、蜂鸣器状态
- 关机按钮：系统关机确认

**摄像头管理**：
- 页面显示时自动启动摄像头（`on_show`）
- 切换到其他页面时自动释放摄像头（`stop_camera`）

### 5.2 密码页（PasswordPage）

**功能**：数字键盘密码输入界面

**核心组件**：
- 密码输入框（显示●）
- 4×3数字键盘（含删除和确认）
- 验证结果提示

**交互流程**：

```
输入密码 → 点击确认 → check_door_password() → 密码正确 → 继电器开启3秒 → 返回主页
                                          → 密码错误 → 提示错误 → 清空输入框
```

### 5.3 指纹页（FingerprintPage）

**功能**：指纹开锁界面（占位实现）

**当前状态**：仅显示标题和返回按钮，等待硬件接入

### 5.4 卡片页（CardPage）

**功能**：卡片开锁界面（占位实现）

**当前状态**：仅显示标题和返回按钮，等待NFC/RFID模块接入

### 5.5 管理页（AdminPage）

**功能**：管理员登录和设备管理

**双状态设计**：

1. **登录状态**：数字键盘密码输入
2. **管理状态**：
   - 门锁控制（开门/关门）
   - 修改门锁密码
   - 人脸库管理（采集/训练/检验）

**人脸库管理功能**：

| 功能 | 实现方法 |
|------|----------|
| 新增人物 | 创建 `dataset/{name}/` 文件夹 |
| 采集人脸 | 调用 `face_recognizer.capture_faces()` |
| 训练特征库 | 调用 `face_recognizer.build_database()` |
| 实时检验 | 调用 `face_recognizer.test_realtime()` |
| 图片检验 | 调用 `face_recognizer.recognize_image()` |

## 九、已知问题

### 9.1 接口不匹配

`admin_page.py` 中调用了 `face_recognizer` 的以下方法，但 `face_recognizer.py` 中实际并未实现：

| admin_page.py 调用的方法 | face_recognizer.py 实际方法 | 状态 |
|-------------------------|---------------------------|------|
| `capture_faces(name)` | `add_face_from_camera(name)` | ❌ 方法名不一致 |
| `build_database()` | 无对应方法 | ❌ 未实现 |
| `test_realtime()` | 无对应方法 | ❌ 未实现 |
| `recognize_image(path)` | 无对应方法 | ❌ 未实现 |

**解决方案**：

方案一：在 `face_recognizer.py` 中补充缺失的方法，实现人脸采集、训练、检验功能

方案二：修改 `admin_page.py`，将调用改为 `face_recognizer.py` 中已有的方法

### 9.2 预留页面未集成

`pages/face_page.py`（FacePage）已创建但未在 `gui_main.py` 中注册和集成，属于预留页面，当前不参与应用流程。

### 9.3 依赖版本兼容

`face_recognition` 库依赖 `dlib`，在树莓派上编译安装较复杂，建议使用 OpenCV SFace 方案替代（见人脸识别文件夹）。
