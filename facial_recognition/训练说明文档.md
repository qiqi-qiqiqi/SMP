# 人脸识别门禁系统 - 训练说明文档

## 一、项目概述

本项目是一个基于 **OpenCV DNN** 和 **预训练ONNX模型** 的轻量级人脸识别门禁系统，运行于树莓派平台。系统采用 **特征提取+余弦相似度匹配** 的方案，无需从头训练深度学习模型，而是利用已训练好的人脸检测和识别模型进行特征提取，构建人脸特征库用于实时身份验证。

#### 步骤二：数据增强

代码位置：[augment_image](file:///f:/book/就业/项目/树莓派/门禁/人脸识别/一体化脚本.py#L94-L111)

```python
def augment_image(img):
    augmented = [img]                    # 原图
    augmented.append(cv2.flip(img, 1))   # 水平翻转
    for alpha in [0.85, 1.15]:           # 对比度调整
        adjusted = cv2.convertScaleAbs(img, alpha=alpha, beta=0)
        augmented.append(adjusted)
    for beta in [-20, 20]:               # 亮度调整
        adjusted = cv2.convertScaleAbs(img, alpha=1.0, beta=beta)
        augmented.append(adjusted)
    for angle in [-10, 10]:              # 旋转±10°
        M = cv2.getRotationMatrix2D((cols/2, rows/2), angle, 1)
        rotated = cv2.warpAffine(img, M, (cols, rows), ...)
        augmented.append(rotated)
    # 对比度+亮度组合调整
    for alpha, beta in [(0.9, -10), (1.1, 10)]:
        adjusted = cv2.convertScaleAbs(img, alpha=alpha, beta=beta)
        augmented.append(adjusted)
    return augmented  # 返回10张增强图片
```

**增强策略汇总**：

| 变换类型 | 参数 | 数量 | 作用 |
|---------|------|------|------|
| 原图 | - | 1 | 基准图片 |
| 水平翻转 | `flipCode=1` | 1 | 增加姿态多样性 |
| 对比度降低 | `alpha=0.85` | 1 | 适应低对比度场景 |
| 对比度增强 | `alpha=1.15` | 1 | 适应高对比度场景 |
| 亮度降低 | `beta=-20` | 1 | 适应低光照环境 |
| 亮度增强 | `beta=20` | 1 | 适应强光环境 |
| 逆时针旋转 | `-10°` | 1 | 适应头部微小偏转 |
| 顺时针旋转 | `10°` | 1 | 适应头部微小偏转 |
| 组合调整1 | `alpha=0.9, beta=-10` | 1 | 综合适应能力 |
| 组合调整2 | `alpha=1.1, beta=10` | 1 | 综合适应能力 |

**效果**：1张原图 → **10张增强图片**，显著提高特征库的鲁棒性。

aug_feat = aug_feat / np.linalg.norm(aug_feat)

**SFace模型原理**：
- 基于SFace提出的自适应间隔损失函数（adaptive margin loss）训练
- 输出128维特征向量（实际验证：(1, 128)）
- 特征空间具有良好的可分性
- 通过自适应调整类间间隔，在保持类内紧凑性的同时最大化类间区分性

**人脸对齐作用**：
- 将检测到的人脸根据关键点旋转、缩放、平移到标准位置
- 消除姿态、光照、角度的影响
- 确保特征提取的一致性

## 三、核心原理

### 3.1 人脸识别原理

本系统采用 **度量学习（Metric Learning）** 方案：

```
原始图片 → 人脸检测 → 人脸对齐 → 特征提取 → 128维特征向量
                                                    ↓
                                            余弦相似度计算
                                                    ↓
                                            阈值判断 → 身份确认
```

### 3.2 余弦相似度匹配

代码位置：[camera_realtime_recognize](file:///f:/book/就业/项目/树莓派/门禁/人脸识别/一体化脚本.py#L479-L485)

```python
# 计算待识别特征与特征库中所有特征的余弦相似度
sims = np.dot(person_feats, feat)  # 向量内积 = 余弦相似度（已归一化）

# 取Top-K相似度的平均值
top_k_sims = np.sort(sims)[-TOP_K:]  # TOP_K = 5
avg_sim = float(np.mean(top_k_sims))
```

**数学原理**：

对于两个归一化向量 **A** 和 **B**：

$$
\text{cosine\_similarity}(A, B) = \frac{A \cdot B}{\|A\| \|B\|} = A \cdot B
$$

由于特征向量已进行L2归一化（$\|A\| = \|B\| = 1$），余弦相似度等于向量内积。

**阈值设定**：

| 相似度范围 | 含义 | 系统判定 |
|-----------|------|----------|
| [0.65, 1.0] | 匹配成功 | 显示姓名和相似度 |
| [0.0, 0.65) | 匹配失败 | 显示"未知" |

### 3.3 模型文件说明

| 文件 | 模型名称 | 来源 | 作用 |
|------|----------|------|------|
| `face_detection_yunet_2023mar.onnx` | YuNet | OpenCV官方模型库 | 人脸检测，输出人脸框和关键点 |
| `face_recognition_sface_2021dec_int8.onnx` | SFace (INT8量化) | OpenCV官方模型库 | 人脸特征提取，输出128维特征 |

**INT8量化优势**：
- 模型体积更小（约4MB）
- 推理速度更快（适合树莓派CPU）
- 精度损失极小（在人脸特征提取任务中可忽略）

## 四、与YOLO标定训练的区别

### 4.1 本质差异

| 维度 | 本系统（特征提取+匹配） | YOLO目标检测训练 |
|------|------------------------|------------------|
| **核心任务** | 身份识别（1:N匹配） | 目标检测（类别+位置） |
| **训练方式** | 不训练模型，只构建特征库 | 训练神经网络权重 |
| **标注需求** | 无需标注，只需按人名分类图片 | 需要标注bounding box和类别 |
| **模型更新** | 更新特征库（增量添加） | 重新训练模型（全量训练） |
| **计算资源** | 低（CPU即可运行） | 高（需要GPU训练） |

### 4.2 技术路线对比

```
本系统：
图片 → 预训练检测器 → 预训练特征提取器 → 特征向量 → 余弦匹配

YOLO训练：
标注数据(图片+XML/TXT) → 训练YOLO网络 → 权重文件 → 推理检测
```

### 4.3 适用场景对比

| 场景 | 本系统 | YOLO |
|------|--------|------|
| 人脸识别门禁 | ✅ 推荐 | ❌ 不适用 |
| 通用目标检测 | ❌ 不适用 | ✅ 推荐 |
| 小样本学习 | ✅ 优秀（少量图片即可） | ❌ 需要大量标注数据 |
| 实时性要求高 | ✅ 满足（树莓派可达10+FPS） | ⚠️ 需要优化 |

### 4.4 关键区别总结

**本系统的核心不是"训练神经网络"，而是"构建和维护特征库"**：

1. **无反向传播**：不涉及梯度下降和模型权重更新
2. **无损失函数**：不需要定义损失函数来优化模型
3. **增量更新**：新增人员只需添加新文件夹，重新运行训练即可
4. **数据依赖性低**：每人只需10~50张照片即可获得良好效果

## 五、相关知识点

### 5.1 人脸检测算法

| 算法 | 特点 | 本项目使用 |
|------|------|-----------|
| YuNet | 轻量级、高精度、ONNX部署 | ✅ |
| MTCNN | 多阶段检测、关键点定位 | ❌ |
| Haar级联 | 传统方法、速度快、精度低 | ❌ |

### 5.2 人脸特征提取算法

| 算法 | 特征维度 | 损失函数 | 本项目使用 |
|------|----------|----------|-----------|
| SFace | 128维 | 自适应间隔损失（adaptive margin loss） | ✅ |
| ArcFace | 512维 | ArcFace（加性角度间隔损失） | ❌ |
| FaceNet | 128维 | Triplet Loss | ❌ |

### 5.3 度量学习损失函数

| 损失函数 | 特点 |
|----------|------|
| **Triplet Loss** | 最小化同类距离，最大化异类距离 |
| **ArcFace** | 在角度空间中最大化类间角度差异 |
| **CosFace** | 余弦边际损失，增强类间区分性 |

### 5.4 数据增强技术

本项目使用的增强方法属于 **几何变换** 和 **像素变换**：

- **几何变换**：翻转、旋转
- **像素变换**：亮度调整、对比度调整

### 5.5 特征匹配算法

| 算法 | 计算方式 | 本项目使用 |
|------|----------|-----------|
| **余弦相似度** | 向量内积（归一化后） | ✅ |
| 欧氏距离 | $\sqrt{\sum (a_i - b_i)^2}$ | ❌ |
| 曼哈顿距离 | $\sum \|a_i - b_i\|$ | ❌ |

## 六、关键点总结

### 6.1 训练阶段关键点

1. **图片质量控制**：
   - 人脸占比适中（建议图片中人脸宽度≥100像素）
   - 光线均匀，避免过曝或欠曝
   - 正面人脸，角度偏差≤30°

2. **数据增强策略**：
   - 增强倍数不宜过多（当前10倍较为合适）
   - 增强幅度不宜过大（旋转≤10°，亮度±20）

3. **特征归一化**：
   - 必须进行L2归一化，否则余弦相似度计算不准确

4. **增量训练**：
   - 新增人员后重新运行训练，会自动合并到特征库

### 6.2 识别阶段关键点

1. **阈值选择**：
   - `SIM_THRESHOLD = 0.65` 为经验值
   - 实际使用中应根据测试效果调整（范围建议0.6~0.75）

2. **Top-K策略**：
   - `TOP_K = 5` 取相似度最高的5个特征求平均
   - 有效减少单张低质量特征的影响

3. **实时性优化**：
   - 使用INT8量化模型加速推理
   - 限制最大检测人脸数量（`maxNumFaces=10`）

### 6.3 部署关键点

1. **模型文件传递**：
   - 将 `face_db.npy` 和 `names.npy` 复制到树莓派
   - 确保文件路径与代码中定义的一致

2. **依赖安装**：
   ```bash
   pip install opencv-python numpy
   ```

3. **树莓派性能优化**：
   - 关闭不必要的后台进程
   - 使用较低分辨率（如640×480）提高帧率
