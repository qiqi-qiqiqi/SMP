# 智慧门禁小程序 - 说明文档

## 一、项目概述

本项目是基于 **微信小程序** 构建的智慧门禁远程监控与控制应用，通过 **华为云IoT平台** 实现与树莓派门禁设备的远程通信。小程序采用 **双通道通信架构**：REST API 用于可靠的状态查询和命令下发，MQTT 用于实时数据推送，实现了环境监控、设备控制、实时通信和操作日志等核心功能。

### 3.3 MQTT实时通信（核心技术亮点）

#### 手动MQTT协议实现

本项目最核心的技术亮点是 **手动构建MQTT二进制协议包**，而非使用现成的MQTT客户端库。这是因为微信小程序环境限制，无法直接使用标准MQTT库，因此采用WebSocket封装MQTT协议。

##### CONNECT包构建

代码位置：[mqttApi.js](file:///f:/book/就业/项目/树莓派/门禁/小程序/接口实现/mqttApi.js#L62-L129)

```javascript
function sendConnectPacket(username, password, clientId) {
  const packet = []
  
  const protoName = 'MQTT'      // 协议名称
  const protoVersion = 4        // MQTT 3.1.1
  const flags = 0xC2            // 连接标志（用户名+密码+保持连接）
  const keepalive = 60          // 心跳间隔（秒）
  
  // 构建Payload
  const payload = []
  
  // 协议名称长度 + 名称
  payload.push((protoName.length >> 8) & 0xFF)
  payload.push(protoName.length & 0xFF)
  for (let i = 0; i < protoName.length; i++) {
    payload.push(protoName.charCodeAt(i))
  }
  
  // 协议版本
  payload.push(protoVersion)
  
  // 连接标志 (0xC2 = 11000010)
  // bit7: 用户名标志=1, bit6: 密码标志=1, bit1: 保持连接标志=1
  payload.push(flags)
  
  // 保持连接时间 (大端序)
  payload.push((keepalive >> 8) & 0xFF)
  payload.push(keepalive & 0xFF)
  
  // Client ID
  payload.push((clientId.length >> 8) & 0xFF)
  payload.push(clientId.length & 0xFF)
  for (let i = 0; i < clientId.length; i++) {
    payload.push(clientId.charCodeAt(i))
  }
  
  // Username 和 Password（同上格式）
  // ...
  
  // 编码剩余长度（可变长度编码）
  let remainingLength = payload.length
  const remainingEncoded = []
  do {
    let byte = remainingLength % 128
    remainingLength = Math.floor(remainingLength / 128)
    if (remainingLength > 0) {
      byte |= 0x80  // 最高位表示后续还有字节
    }
    remainingEncoded.push(byte)
  } while (remainingLength > 0)
  
  // 构建完整包
  packet.push(0x10)              // CONNECT包类型 (0001 0000)
  packet.push(...remainingEncoded)
  packet.push(...payload)
  
  const buffer = new Uint8Array(packet)
  ws.send({ data: buffer })
}
```

**MQTT包结构解析**：

```
┌────────────┬─────────────────┬────────────────────────────────────┐
│  固定头    │   可变头        │           Payload                 │
│  (2+ bytes)│   (N bytes)     │           (M bytes)               │
├────────────┼─────────────────┼────────────────────────────────────┤
│ 0x10       │ Protocol Name   │ Client ID (可选)                  │
│ (CONNECT)  │ Protocol Version│ Username (可选)                   │
│            │ Connect Flags   │ Password (可选)                   │
│            │ Keepalive       │ Will Topic (可选)                 │
│            │                 │ Will Message (可选)               │
└────────────┴─────────────────┴────────────────────────────────────┘
```

##### 剩余长度编码（Variable Byte Integer）

```javascript
do {
  let byte = remainingLength % 128
  remainingLength = Math.floor(remainingLength / 128)
  if (remainingLength > 0) {
    byte |= 0x80  // 最高位置1，表示后续还有字节
  }
  remainingEncoded.push(byte)
} while (remainingLength > 0)
```

**编码规则**：
- 每个字节的低7位表示数值
- 最高位为1表示后续还有字节
- 最高位为0表示这是最后一个字节

**示例**：长度128 → `0x81 0x00`（128 = 1×128 + 0）

##### 消息处理逻辑

代码位置：[mqttApi.js](file:///f:/book/就业/项目/树莓派/门禁/小程序/接口实现/mqttApi.js#L131-L176)

```javascript
function handleMQTTMessage(data) {
  const buffer = new Uint8Array(data)
  const packetType = (buffer[0] >> 4) & 0x0F  // 提取高4位作为包类型
  
  if (packetType === 2) {
    // CONNACK - 连接确认
    const returnCode = buffer[3]
    if (returnCode === 0) {
      isConnected = true
      startPing()
      onConnectCallback()
    }
  } else if (packetType === 3) {
    // PUBLISH - 收到消息
    
    // 解析主题
    const topicLength = (buffer[2] << 8) | buffer[3]
    let topic = ''
    for (let i = 4; i < 4 + topicLength; i++) {
      topic += String.fromCharCode(buffer[i])
    }
    
    // 解析Payload
    let payloadStart = 4 + topicLength
    if (buffer[0] & 0x01) {  // QoS > 0 时有Packet ID
      payloadStart += 2
    }
    const payload = buffer.slice(payloadStart)
    const message = new TextDecoder().decode(payload)
    
    onMessageCallback(topic, message)
  } else if (packetType === 4) {
    // PINGRESP - 心跳响应
    console.log('[MQTT] PINGRESP收到')
  }
}
```

**包类型对照表**：

| 包类型 | 十六进制 | 含义 |
|--------|----------|------|
| CONNECT | 0x10 | 客户端连接请求 |
| CONNACK | 0x20 | 连接确认 |
| PUBLISH | 0x30 | 消息发布 |
| PUBACK | 0x40 | 发布确认 |
| SUBSCRIBE | 0x80 | 订阅请求 |
| SUBACK | 0x90 | 订阅确认 |
| PINGREQ | 0xC0 | 心跳请求 |
| PINGRESP | 0xD0 | 心跳响应 |
| DISCONNECT | 0xE0 | 断开连接 |

### 3.4 Token认证与缓存

#### Token获取流程

代码位置：[huaweiCloudApi.js](file:///f:/book/就业/项目/树莓派/门禁/小程序/接口实现/huaweiCloudApi.js#L8-L49)

```javascript
let cachedToken = null
let tokenExpireTime = 0

function getToken(callback) {
  const now = Date.now()
  
  // 检查缓存是否有效（1小时有效期）
  if (cachedToken && now < tokenExpireTime) {
    callback(null, cachedToken)
    return
  }
  
  wx.request({
    url: `${iamEndpoint}/v3/auth/tokens`,
    method: 'POST',
    header: { 'Content-Type': 'application/json' },
    data: {
      auth: {
        identity: {
          methods: ['password'],
          password: {
            user: { 
              name: iamUser, 
              password: iamPwd, 
              domain: { name: accountName } 
            }
          }
        },
        scope: { project: { name: region } }
      }
    },
    success: (res) => {
      if (res.statusCode === 200 || res.statusCode === 201) {
        cachedToken = res.header['X-Subject-Token']
        tokenExpireTime = now + 3600 * 1000  // 1小时后过期
        callback(null, cachedToken)
      }
    }
  })
}
```

**华为云IAM认证流程**：

```
POST /v3/auth/tokens
{
  "auth": {
    "identity": {
      "methods": ["password"],
      "password": {
        "user": {
          "name": "用户名",
          "password": "密码",
          "domain": { "name": "账号名" }
        }
      }
    },
    "scope": { "project": { "name": "区域" } }
  }
}

响应头: X-Subject-Token: <token>
```

**缓存机制**：
- Token存储在内存中（`cachedToken`）
- 设置1小时过期时间（`tokenExpireTime`）
- 过期后自动重新获取
- 请求失败时清空缓存，下次重新获取

```javascript
      }
      const { temp, humi, relay, beep, rain, person } = parseDeviceData(data)
      this.setData({ temp, humi, relay, beep, rain, person, lastUpdate: this.formatTime() })
    })
  }, 5000)  // 每5秒刷新一次
}
```

**设计特点**：
- 定时器存储在页面实例中（`this.timer`）
- 页面卸载时自动清理（`onUnload`）
- 刷新间隔5秒，平衡实时性和服务器压力

## 五、安全注意事项

### 5.1 硬编码凭证（严重风险）

**问题**：[config.js](file:///f:/book/就业/项目/树莓派/门禁/小程序/接口实现/config.js) 中包含明文的华为云账号凭证：

```javascript
module.exports = {
  accountName: 'CSDN-w40624_cc',
  iamUser: 'CSDN-w40624_cc',
  iamPwd: '2256792214*',  // 明文密码！
  mqttPassword: 'b051a0a8ad08504f1122a589d1a8b40d95f0a7253cbaf6eec13c92d7c970b68a'
}
```

**风险等级**：🔴 高风险

**建议解决方案**：

1. **使用后端代理**：小程序不直接存储凭证，通过自有后端服务转发请求
2. **动态获取凭证**：通过OAuth2等方式动态获取临时凭证
3. **加密存储**：对敏感信息进行加密后存储，运行时解密

### 5.2 网络传输安全

- MQTT使用 `wss://`（WebSocket Secure），数据传输加密
- REST API使用 `https://`，数据传输加密
- Token有效期1小时，降低泄露风险

## 七、已知问题

### 7.1 Token过期处理不完善

当Token过期且重新获取失败时，后续请求会一直失败，需要手动刷新页面恢复。

### 7.2 MQTT重连逻辑简单

当前重连间隔固定为5秒，没有指数退避机制，网络不稳定时可能造成服务器压力。

### 7.3 日志存储容量限制

日志仅保留最近20条，且存储在本地，无法远程查看历史日志。

### 7.4 缺少错误重试机制

REST API请求失败后没有重试逻辑，可能导致命令下发丢失。
